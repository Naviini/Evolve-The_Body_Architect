3.1 Introduction

This chapter narrates the journey from concept to working prototype across an Agile sprint cadence combined with Design Science Research (DSR), constructing and reflecting upon an artefact (Hevner et al., 2004). Scrum-style iterations preserved responsiveness while maintaining thesis-grade traceability between literature claims and engineered modules.

3.2 High-Level Architecture

The solution adopts a three-tier Client–Server–AI architecture grounded in the shipped codebase.

The Client tier is an Expo / React Native application responsible for navigation, capture pipelines (camera / gallery), gamification UI, and deterministic wellness engines implemented in TypeScript.

Persistent interaction state uses Expo SQLite as the authoritative store during demonstration and offline-first flows: onboarding snapshots, meal logs, workout sessions, rewards, and coach conversation turns include synchronisation metadata suitable for incremental mirroring.

The Server tier uses Supabase PostgreSQL with authentication hooks (where configured) and Row Level Security policies isolating user-owned tuples — aligning privacy posture with GDPR-oriented expectations for sensitive wellness narratives.

The AI tier is deliberately hybrid rather than monolithic: (1) multimodal meal estimation uses OpenAI GPT-4o Vision through chat completions with JSON shaping prompts for reproducible parsing; (2) conversational coaching uses GPT-4o-mini with streak-aware conditioning drawn from SQLite aggregates and guarded refusal pathways for crisis language; (3) optional Gemini 2.0 Flash vision supports auxiliary body-photo commentary behind feature flags; (4) a parallel ViT Food-101 fine-tuning stack lives under calorie-tracker/model with scripts for dataset preparation, fine-tuning, evaluate_model.py metrics export, optional ONNX export, and FastAPI inference_server.py for reproducible classification benchmarking distinct from the multimodal meal UX path.

This separation documents an academically honest engineering compromise: dense convolutional / transformer classifiers excel at closed-set benchmarks, whereas multimodal LLMs trade calibrated taxonomy constraints for descriptive specificity on culturally heterogeneous plates — the artefact presents both.

3.3 Dataset and Data Collection

Food-101 (Bossard et al., 2014) remains the backbone supervised corpus for the ViT pathway owing to scale and citation-grade comparability.

Regional realism for Sri Lanka is pursued along two complementary axes aligned with repository artefacts: (i) catalogue-oriented enrichment via scripted nutrition seeds suitable for manual logging fidelity (seed-srilankan-foods.mjs pattern); (ii) multimodal prompting emphasising South Asian dish specificity within scan.service.ts rather than relying solely on Western-centric label vocabularies.

Onboarding captures extend across biological markers, lifestyle rhythms, dietary preference vectors, stress proxies, family histories, and circumferences — intentionally exceeding shallow calorie-apps demographics to operationalise literature-motivated metabolic nuance.

Requirements elicitation combined structured interviews, competitor audits (MyFitnessPal, Fitbit-class ecosystems), and survey snapshots consistent with IEEE SRS guidance (IEEE, 2014).

3.4 Data Pre-Processing (ViT Track)

Training augmentations follow established ViT transfer protocols implemented in model/prepare_dataset.py orientation: stochastic resized crops, flips, mild rotations, photometric jitter, light affine perturbations, and occasional greyscale drops supporting domestic lighting variability.

Evaluation retains deterministic resize + centre crop consistent with ViT processor geometry.

3.6 Model and Logic Development

ViT selection rationale persists: global self-attention supports spatial mixing cues theorised as advantageous on heterogeneous food plating relative to strictly local convolution inductive biases (Dosovitskiy et al., 2020; Liu et al., 2025). Hyper-parameters documented previously remain representative design defaults executed inside Google Colab GPU notebooks bundled as reproducible scripts/notebooks.

The Metabolic Intelligence Engine remains the symbolic-regression-style fusion emphasised academically: Layer 1 structural ectomorphy via Heath–Carter height–weight ratio transforms; Layer 2 composition via Hodgdon–Beckett US Navy circumference body-fat estimation augmented by wrist-to-height frame biasing when measurements exist; Layer 3 lifestyle classification scoring driven by activity, occupational demand, and exercise cadence; Layer 4 metabolic modulation integrating diet archetype, sleep debt proxies, stress sliders, blood marker flags, and family risk motifs.

Absent waist + neck circumferences, weights honourably degrade composition contribution while reallocating influence across structural, lifestyle, and metabolic strata — documented inline within src/lib/bodyTypeEngine.ts as the Implemented analogue of earlier prose weights.

Calorie logic chains Mifflin–St Jeor basal metabolic rate through composite total daily energy expenditure multipliers before clamped recommendation ceilings consistent with conservative wellness tooling defaults.

Meal ideas synthesise through deterministic template allocation inside dietPlanEngine.ts: macro envelopes derive from personalised calorie targets and diet archetypes (ketogenic bias, plant-forward splits, allergy tagging), while cuisine affinity blends nationality heuristics with explicit preference arrays — yielding inspectable textual rationales suitable for academic critique unlike opaque unconstrained text-only generation.

Workouts instantiate templated weekly structures filtered by equipment declarations and conservative pregnancy/injury avoidance tagging consistent with responsible MVP framing.

3.7 Validation Strategy

ViT accuracy claims must cite outputs emitted by evaluate_model.py once trained_model weights exist — Top-1 / Top-5 / confusion diagnostics constitute primary quantitative ML evidence.

Multimodal scanning validates along usability dimensions: structured JSON integrity, alternative hypothesis enumeration, graceful deterministic demo fallback without secrets for graders, and qualitative reviewer judgement on culturally labelled plates.

Somatotype logic validates through deterministic recomputation checks against Navy reference calculators on synthetic measurement tuples plus regression awareness when circumferences absent.

Future strengthened evaluation could incorporate labelled regional imagery corpus extension and prospective IRB-governed anthropometric correlation studies — explicitly out of scope unless ethically cleared.

3.8 System Development Timeline

Sprints progressed requirements → modelling → implementation → testing → documentation mirroring earlier narrative.

Architectural pivot justification emphasises Expo iteration velocity, Supabase relational ergonomics for rich profiles, and pragmatic multimodal API orchestration from TypeScript while preserving standalone ViT reproducibility for scholarly benchmarking.
